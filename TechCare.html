Tech.Care

Overview

In 2024, I completed a front-end development skills test for the company Coalition Technologies. This test contained developing a webpage based on the provided design in Adobe XD and using the patient data API and Postman to display the already inputted data. Using the provided materials along with HTML, CSS, and JavaScript I created the dashboard for Tech.Care, a healthcare website where patients can view their medical data.

Tools

Based on the requirements, I concluded on using the tools:

  • Visual Studio Code
  • HTML
  • CSS
  • JavaScript
  • Postman
  • Patient Data API
  • Chart.js

My Role

As the Front-End Developer, my role contained the following:

  • Creating the webpage inputs from scratch using the prototype/wireframe
  • Designing the layout and colors using the exact specifics from Adobe XD
  • Utilizing the API to input data from a back-end so it can be easily updated and adjusted
  • Comparing and adjusting my work to the prototype/wireframe design to maintain consistency
  • Ensuring cybersecurity by keeping the basic auth encrypted

Chart.js

One of the main focuses of the dashboard page was a responsive graph that adjusted for each patient depending on their specific data. To do this, it was recommended we use Chart.js. Chart.js is an open source JavaScript library used to make graphs and other various displays on the front end and also comes with built-in chart types and configurations. Using this library, I was able to make a modern and responsive line graph with points that documented the blood pressure history of the patient over a period of time and displayed the current values on the key. This process also involved many steps relating to the design of the graph, including backgroundColor, borderCapStyle, borderJoinStyle, etc.

Problem

While most of the dashboard was created from scratch only using the specifications and images from Adobe XD, the patent data was inputted using the provided API in order to not hard code patient information so it can easily be adjusted. While implementing this API into the system using Postman, it was revealed this API used Basic Auth, which is a way to provide a web browser the username and password needed to show the data. While I have used basic auth before, it has only been in personal or school projects where cybersecurity was not an issue, so I needed to find a way to not hard code the basic auth key and encrypt it myself using the proper methods.

Solution

After some research and trial and error using various methods, I concluded the best way to not hard code the basic auth key and encrypt it myself was to add the const authHeaderValue and assign it 'Basic encrypted username:password'. Then to get the authorization needed, use Authorization: authHeaderValue. This method encrypts the basic auth credentials while still being able to use them to pull the necessary patient data from the API.

Next Steps

While I have completed this test, a step I would take next time is increasing responsiveness.

Although I did follow everything on the Adobe XD wireframe (including border-radius, padding, and margin pixel sizes and exact color hex codes), I wish I could have improved this design by making it more responsive. The issue in following these exact instructions was that pixel sizes were hard-coded and made to fit only a select number of screen sizes. It is due to this that while the webpage originally looked great on my monitor, it got squished and rearranged improperly on my laptop's screen. I made a few minor changes mainly using @media in the CSS to resize the content boxes to make them fit evenly, but because this test was not originally built to be responsive, this only went so far.

Learnings

Overall, this experience taught me a lot about converting a wireframe to developed code and using an API. This was my first major time using an API to implement back-end data to a front-end project outside of class, and it was really interesting to put my knowledge to the test. Despite the struggles at the beginning, it was all worth it when the success of it all coming together would work and filled me with even more motivation to continue. Because I am used to working on projects of this magnitude in a team, this experience helped me grow my skills as a developer and increased my resilience and self reliance as a coder. Though the project has been completed, I am proud of the effort I put in and believe that it shows my skill and determination.
